# Nav tweaks, Ollama Cloud provider, shorter API keys — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Four small independent changes — API keys moved under Dashboard in the nav, a collapsible "// Manage" nav group, a first-class Ollama Cloud external provider, and shorter (37-char, same-entropy) `dpk_` API keys.

**Architecture:** Console-only changes for the two nav items (`console/app/nav.tsx` + one CSS block). The Ollama provider mirrors the existing OpenRouter preset across three files (console preset, CP provider table, gateway config prefix + api_base fallback). Key shortening is a new pure `secretToken()` generator in `control-plane/src/id.ts` used by `repo.ts createApiKey` — gateway auth is sha256-lookup-only, so old keys keep working and nothing else changes.

**Tech Stack:** Next.js (console, production builds only), Fastify/TS control plane (Node test runner, live dev Postgres), LiteLLM gateway config generated by `gateway-config.ts`.

**Spec:** `docs/superpowers/specs/2026-07-17-nav-ollama-short-keys-design.md`

## Global Constraints

- Console is ALWAYS verified with a production build: `cd console && npx next build` (dev mode banned as too slow). If a console process is running under `next start`, a rebuild requires a restart before live checks (stale chunk hashes cause client-side exceptions).

- Throwaway test workspaces MUST use a swept name prefix, e.g. `t-<tag>-${Date.now()}` (see `scripts/sweep-workspaces.ts` TEST_WS_PATTERN).
- UI rules: no transparent text buttons in general — the nav group toggle is a disclosure control in the tabs/quiet-controls family and must look exactly like today's group label (no button chrome).
- This repo is CRLF; after bulk line deletions via Edit, re-check `git diff` for accidental line joins.

```tsx
import { usePathname } from "next/navigation";

import { Mark } from "./lib/mark";
import { WorkspacePicker } from "./lib/ws-picker";

const GROUPS: { title: string | null; collapsible?: boolean; items: [string, string, IconName][] }[] = [
  { title: null, items: [["Dashboard", "/", "dashboard"], ["API keys", "/api-keys", "key"]] },
  {
    title: "Managed Agents",
    items: [
      ["Agents", "/agents", "agent"],
      ["Skills", "/skills", "skill"],
      ["Sessions", "/sessions", "session"],
      ["Environments", "/environments", "env"],
      ["Credential vaults", "/vaults", "vault"],
      ["Files", "/files", "file"],
      ["Memory stores", "/memory-stores", "memory"],
    ],
  },
  { title: "Serving", items: [["Model catalog", "/catalog", "catalog"], ["Deployments", "/deployments", "deploy"], ["Routings", "/routings", "routing"], ["Pools", "/pools", "pool"], ["Cache", "/cache", "cache"]] },
  { title: "Manage", collapsible: true, items: [["Usage - API", "/usage/api", "usage"], ["Usage - Sessions", "/usage/sessions", "usage"], ["Settings", "/settings", "settings"], ["Workspaces", "/workspaces", "workspace"]] },
];

export function Nav({ workspaces, current }: { workspaces: { id: string; name: string; status: string }[]; current: string }) {
  const path = usePathname();
  const isActive = (href: string) => (href === "/" ? path === "/" : path === href || path.startsWith(href + "/"));
  // Collapsible groups: closed by default, forced open while a child route is
  // active (the active item must never be hidden). Plain state — survives
  // client-side navigation (Nav stays mounted), resets on hard reload.
  const [openGroups, setOpenGroups] = useState<Record<string, boolean>>({});

  return (
    <nav className="sidebar">
      <Link href="/" className="brand">
        <Mark className="brand-mark" />
        <span className="brand-text">DEVPROOF<span className="ai">.AI</span><span className="cp">OWN YOUR SCALABLE AI</span></span>
      </Link>
      <div className="ws">
        <label>Workspace</label>
        <WorkspacePicker workspaces={workspaces} current={current} />
      </div>
      {GROUPS.map((g, i) => {
        const expanded = !g.collapsible || !!openGroups[g.title!] || g.items.some(([, href]) => isActive(href));
        return (
          <div key={i}>
            {g.title && (g.collapsible ? (
              <button
                type="button"
                className="group group-toggle"
                aria-expanded={expanded}
                onClick={() => setOpenGroups((s) => ({ ...s, [g.title!]: !expanded }))}
              >
                {g.title}
                <span className={`group-chevron ${expanded ? "open" : ""}`} aria-hidden>▸</span>
              </button>
            ) : (
              <div className="group">{g.title}</div>
            ))}
            {expanded && g.items.map(([label, href, icon]) => {
              const I = Icon[icon];
              return (
                <Link key={href} href={href} className={`item ${isActive(href) ? "active" : ""}`}>
                  <I /><span>{label}</span>
                </Link>
              );
            })}
          </div>
        );
      })}
    </nav>
  );
}
```

- [ ] **Step 3: Implement secretToken and use it in createApiKey**

Append to `control-plane/src/id.ts`:

```ts
const B62 = "0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz";

/** Uniform base62 secret token, rejection-sampled from crypto randomBytes
 *  (bytes ≥ 248 are rejected; 248 = 4·62, so accepted bytes are uniform
 *  mod 62 — no modulo bias). 33 chars ≈ 196.5 bits, ≥ the 192 bits of the
 *  legacy dpk_ 48-hex format. */
export function secretToken(len: number): string {
  let out = "";
  while (out.length < len) {
    for (const b of randomBytes(len)) {
      if (b < 248 && out.length < len) out += B62[b % 62];
    }
  }
  return out;
}
```
