# Devproof AI — Sub-plan B: Devproof Operator Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Go operator with `ModelPool` and `ModelDeployment` CRDs (`serving.devproof.ai/v1alpha1`) that reconciles a ModelDeployment into LLMkube `Model` + `InferenceService` resources, applying the pool's nodeSelector, and reports status back. Verified against the live docker-desktop cluster with the sub-plan-A smoke test kept green.

**Architecture:** Hand-rolled controller-runtime project (no kubebuilder CLI — it ships no Windows binary). LLMkube resources are manipulated as `unstructured.Unstructured` so we don't import their pre-1.0 Go module (version tolerance per concept §5.4). The operator runs OUT of cluster (`go run`) for development; containerization arrives with the Helm chart in sub-plan D. Catalog resolution is NOT the operator's job: `ModelDeployment.spec.model` carries source/format denormalized (the control plane resolves catalog → CRD in sub-plan C). This is a deliberate refinement of concept §5.3 — record it in the concept paper changelog when sub-plan C lands.

**Tech Stack:** Go 1.26 (`~/sdk/go/bin/go`), controller-runtime, controller-gen (deepcopy + CRD YAML), Go stdlib `testing`.

## Global Constraints
- Same as sub-plan A (context `docker-desktop`, CPU-only, `devproof-` namespaces, Git Bash scripts, conventional commits).
- Devproof API group: `serving.devproof.ai/v1alpha1`. LLMkube group: `inference.llmkube.dev/v1alpha1`.
- Operator must never delete LLMkube resources it does not own (ownership = label `serving.devproof.ai/owned-by: <modeldeployment-name>` + ownerReference).

```go
// GroupVersion: serving.devproof.ai/v1alpha1
type ModelPoolSpec struct {
    NodeSelector map[string]string `json:"nodeSelector,omitempty"`
    GPUType      string            `json:"gpuType,omitempty"`      // "cpu" allowed
    GPUsPerNode  int32             `json:"gpusPerNode,omitempty"`
    MaxNodes     int32             `json:"maxNodes,omitempty"`
    ScalingMode  string            `json:"scalingMode,omitempty"`  // dynamic|static
}
type ModelSource struct {
    Source string `json:"source"`           // HF resolve URL
    Format string `json:"format"`           // gguf|safetensors
}
type ReplicaBounds struct {
    Min int32 `json:"min"`
    Max int32 `json:"max"`
}
type ModelDeploymentSpec struct {
    Model              ModelSource       `json:"model"`
    CatalogID          string            `json:"catalogId,omitempty"`
    PoolRef            string            `json:"poolRef"`
    Engine             string            `json:"engine,omitempty"`   // auto|llama.cpp|vllm
    Replicas           ReplicaBounds     `json:"replicas"`
    Resources          map[string]string `json:"resources,omitempty"` // cpu, memory, gpu
    TargetTokensPerSec int32             `json:"targetTokensPerSec,omitempty"`
}
type ModelDeploymentStatus struct {
    Phase         string `json:"phase,omitempty"`    // Pending|Deploying|Ready|Failed
    ReadyReplicas int32  `json:"readyReplicas,omitempty"`
    Endpoint      string `json:"endpoint,omitempty"`
    Message       string `json:"message,omitempty"`
}
```
